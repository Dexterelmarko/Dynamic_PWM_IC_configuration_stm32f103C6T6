# Dynamic_PWM_IC_configuration_stm32f103C6T6

[![Platform: STM32](https://img.shields.io/badge/platform-STM32-blue)](https://www.st.com)
[![IDE: STM32CubeIDE](https://img.shields.io/badge/IDE-STM32CubeIDE-green)](https://www.st.com)

## О проекте
Данный проект реализует динамическую конфигурацию портов микроконтроллера stm32f103C6T6 для работы в режиме генерации или измерения ШИМ сигналов. 

Передача команд конфигурации портов и параметров работы осуществляется по интерфейсам USB и UART.

## Возможности
- Генерация ШИМ сигналов с заданными частотой и скважностью
- Измерение ШИМ сигнала с заданными параметрами (размер окна измерений, тип фильтра, время ожидания)
- Просмотр действующей распиновки и конфигурации портов

## Аппаратная часть

### Микроконтроллер
- **Модель:** STM32F103C6T6 (Blue Pill)
- **Тактовая частота:** 72 МГц (внешний кварц 8 МГц)

### Используемые периферийные блоки
- USB (основной интерфейс)
- USART1 (основной интерфейс, скорость 115200 бод)
- SPI1 (для подключения модуля W5500)
- TIM1, TIM2, TIM3 (для конфигурации портов с поддержкой таймера)

## Архитектура проекта

### Описание модулей (Файловая структура)

- **`main.c`** – Инициализация тактирования, HAL, кнопок. Запуск RTOS (если есть).
- **`app_sensors.c`** – Обработка датчиков: опрос, фильтрация данных, ошибки.
- **`comm_uart.c`** – Буферизированный прием/передача по UART, парсинг команд.
- **`protocol.c/h`** – Реализация протокола обмена (см. раздел "Протокол").

### Логическая архитектура (слои)

1. **Hardware Abstraction Layer (HAL/LL)** – Генерация кода CubeMX. Не редактируется.
2. **Board Support Package (BSP)** – Функции `LED_On()`, `Button_Read()`. Работа с пинами.
3. **Application Layer** – Основная логика: `if (temperature > 30) -> relay_on()`.
